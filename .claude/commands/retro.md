# Retro - Retrospective

Generate and fill retrospective template.

## Parse Arguments

`$ARGUMENTS`:
- Empty or `week` - Weekly retro
- `sprint` - Sprint retro (2 weeks)
- `month` - Monthly retro

## 1. Gather Period Data

From work-shell logs:
- Sessions in period
- Decisions made
- Todos completed/pending
- Notes and checkpoints

From git:
- Commits
- PRs merged
- Files changed

## 2. Generate Template

```markdown
# Retrospective: {period}

**기간**: {start_date} ~ {end_date}
**작성일**: {today}

---

## 이번 {period} 요약

### 통계
- 커밋: {commits}개
- 세션: {sessions}개
- 완료한 TODO: {completed_todos}개

### 주요 작업
{major_works_from_logs}

---

## 회고

### 잘한 점 (Keep)
-

### 개선할 점 (Problem)
-

### 시도할 것 (Try)
-

---

## 배운 것

### 기술적
-

### 프로세스
-

---

## 다음 {period} 목표

1.
2.
3.

---

## 메모

{additional_notes}

---

*Generated by work-shell*
```

## 3. Interactive Mode

Use AskUserQuestion to help fill:
- "이번 주 가장 잘한 점은?"
- "개선하고 싶은 점은?"
- "다음 주 목표는?"

## 4. Save

Save to `.work-shell/retros/{date}-{period}.md`

## 5. Output

```
## Retro 생성됨

**기간**: {period}
**파일**: .work-shell/retros/{filename}

### 자동 수집된 데이터
- 커밋: {N}개
- 세션: {N}개
- 결정: {N}개

---

회고 내용을 작성해주세요.
파일을 열어 Keep/Problem/Try 섹션을 채워주세요.
```

User argument: $ARGUMENTS
